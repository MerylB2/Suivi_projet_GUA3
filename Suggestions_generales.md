## Outils Gratuits pour Créer des Schémas et Diagrammes

Je vous demande à tous de produire des diagrammes propres et lisibles dans vos documents de conception. Voici les outils que je recommande, tous gratuits.

---

### Pour les schémas d'architecture et diagrammes techniques

**draw.io — Recommandé en priorité**
Entièrement gratuit, sans compte obligatoire. Utilisable directement en ligne sur draw.io ou en application desktop. Il couvre tout ce dont vous avez besoin : diagrammes d'architecture, flux de données, schémas de base de données, organigrammes. Les fichiers s'exportent en PNG, SVG ou PDF. C'est l'outil que j'attends par défaut dans vos rendus.

Lien : https://app.diagrams.net

---

### Pour les diagrammes UML et la modélisation

**Lucidchart**
Version gratuite limitée à 3 documents — suffisant pour un projet académique. Interface propre et intuitive, idéale si vous débutez avec les diagrammes UML.

Lien : https://lucidchart.com

**PlantUML**
Alternative textuelle : vous écrivez du code simple et le diagramme se génère automatiquement. Très utile pour les diagrammes de séquence et les diagrammes de classes. Recommandé si vous êtes à l'aise avec la syntaxe.

Lien : https://plantuml.com

---

### Pour les schémas de base de données

**dbdiagram.io — Recommandé pour la modélisation BDD**
Fait exactement pour ça. Vous écrivez la structure de votre base en texte simple et le schéma relationnel se génère visuellement. Gratuit, rapide à prendre en main, export PNG/PDF. Je m'attends à ce que vos modèles de données soient produits avec cet outil ou draw.io.

Lien : https://dbdiagram.io

---

### Pour les wireframes et maquettes UI

**Figma**
Gratuit pour les étudiants via le plan Education. C'est l'outil le plus utilisé en entreprise pour les maquettes d'interface. Si vous avez des maquettes à produire pour votre projet, c'est ici.

Lien : https://figma.com/education

**Penpot**
Alternative open source entièrement gratuite, sans limite. Même usage que Figma.

Lien : https://penpot.app

---

### Pour les mind maps et tableaux collaboratifs

**Miro**
Plan gratuit avec 3 tableaux. Très bien pour vos réunions de synchronisation hebdomadaires et vos tableaux Kanban de suivi de projet. Je vous encourage à l'utiliser pour votre tableau À faire / En cours / Terminé.

Lien : https://miro.com

**FigJam**
Intégré à Figma, gratuit et plus léger que Miro. Bien pour les sessions de travail collaboratif à distance.

Lien : https://figma.com/figjam

## Format Attendu pour les Rendus

Je vous rappelle les formats acceptés pour vos documents de conception :

- **Fichiers Markdown (.md)** — pour la documentation technique, les README, les documents de conception
- **PNG ou SVG** — pour les exports de schémas et diagrammes
- **PDF** — pour les rapports finaux

Je n'accepte pas les notebooks Jupyter (.ipynb) comme documents de conception. Un notebook est un outil de développement, pas un document de livraison. Si votre document de conception est actuellement dans un notebook, convertissez-le en Markdown avant le prochain livrable.

---

## Rappel — Ce que J'attends dans Chaque Document de Conception

Quel que soit votre projet, votre document de conception doit contenir au minimum :

- Introduction et contexte
- Objectifs
- Besoins fonctionnels
- Besoins non-fonctionnels
- Contraintes techniques
- Architecture du système avec schéma (produit avec l'un des outils ci-dessus)
- Choix de la stack technique justifié
- Modélisation des données (schéma produit avec dbdiagram.io ou draw.io)
- Endpoints API documentés
- Flux de données
- Structure du projet
- Répartition des rôles (pour les projets en binôme)
- Conclusion

Un document incomplet ou avec des sections vides ne sera pas considéré comme rendu.
